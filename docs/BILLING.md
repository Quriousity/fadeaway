# Fadeaway — 과금 체계 (상용 서비스용 / 미래 계획)

> ⚠️ 현재 앱에 노출된 "과금 모달"은 **상용화 시 적용할 모델을 미리 넣어둔 것**이며,
> 지금 단계에서는 실제 과금이 일어나지 않는다. 구현 우선순위 낮음.

## 기본 원칙

- **모든 기능 10분 무료 · 이후 분당 과금**
- `/분·인` = 분당 이용자 1인 기준 (다자간이면 인원수만큼 합산)

## 요금표

| 기능 | 요금 | 단위 |
|---|---|---|
| 통화 (음성) | 1원 | /분 |
| 영상통화 | 10원 | /분·인 |
| 화면 공유 | 10원 | /분·인 |
| 원격 제어 | 15원 | /분·인 |

## 비용 구조와의 관계

WebRTC는 통화 중 미디어가 P2P로 직접 흐르므로 **서버 변동비는 사실상 TURN 대역폭 하나뿐**
(자세한 내용은 [ARCHITECTURE.md](./ARCHITECTURE.md) 4장 참고).

따라서 위 분당 요금은:
- 통화/영상 자체의 서버 원가가 거의 0인 점을 감안한 **마진 + TURN 변동비 버퍼**
- 광고(콜 오버레이) 외 수익 채널
- 원격 제어가 가장 비싼 이유: 입력 지연 민감 → TURN 의존도/품질 보장 비용이 상대적으로 큼

## 구현 시 고려 (미래)

- 사용량 측정: 통화 시작/종료 타임스탬프 → 분 단위 집계 (Supabase에 기록)
- 10분 무료 차감 로직
- 다자간 인원수 가중 (`·인`)
- 결제 연동 (미정)
